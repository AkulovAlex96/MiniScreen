# MiniScreen — видео из сети (MJPEG по HTTP)

ПК перекодирует любое видео в MJPEG 240x240 (ffmpeg), ESP32-S3 забирает поток
по Wi-Fi, декодирует JPEG-кадры (JPEGDEC) и выводит на дисплей. ~15–20 fps.

```
видео/YouTube/камера → ffmpeg → HTTP :8090 → Wi-Fi → ESP32-S3 → GC9A01
```

## Запуск

### 1. Прошивка

Скопировать [src/secrets.h.example](src/secrets.h.example) в `src/secrets.h` и заполнить:

```cpp
#define WIFI_SSID  "имя_сети"        // 2.4 ГГц
#define WIFI_PASS  "пароль"
#define STREAM_URL "http://192.168.1.100:8090/"   // IP компа
```

(`secrets.h` в .gitignore, в репозиторий не попадает)

IP компа узнать: `ipconfig` (поле IPv4). Прошить: `pio run --target upload`.

### 2. Сервер (нужен ffmpeg в PATH)

```bat
cd server
stream.bat мое_видео.mp4          # файл в цикле, 20 fps
stream.bat видео.mp4 24 5         # 24 fps, качество повыше
stream_youtube.bat "https://youtube.com/watch?v=..."   # нужен yt-dlp
```

Первый запуск: разрешить ffmpeg в файрволе Windows (или открыть порт 8090).

Проверить поток можно браузером/VLC: `http://localhost:8090/` — если картинка
идёт там, но не на ESP, проблема в сети/файрволе.

## Управление

- **BTN1** — вкл/выкл счётчик FPS внизу экрана
- **BTN2** — подсветка

## Как работает приём

- Из HTTP-потока вылавливаются JPEG-кадры по маркерам `FFD8`(начало)/`FFD9`(конец) —
  работает с любым MJPEG-сервером (ffmpeg, mjpg-streamer, IP-камеры)
- Кадр декодируется JPEGDEC в полнокадровый спрайт, вывод одним pushSprite
- Обрыв связи / 5 сек тишины → автопереподключение

## Ограничения и путь к ускорению

- Однопоточная схема: приём → декод → вывод последовательно, ~15–20 fps.
  Разнести приём (core 0) и декод+вывод (core 1) → ~25 fps
- ffmpeg с `-listen 1` держит одного клиента; .bat перезапускает его в цикле
- Звука нет (дисплей же). Можно добавить I2S-усилок и второй поток
