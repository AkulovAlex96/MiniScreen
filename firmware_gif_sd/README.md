# MiniScreen — проигрывание GIF с MicroSD

То же самое, что `firmware_gif`, но без встраивания во flash: гифки читаются
файлами прямо с карты через колбэки `AnimatedGIF` (open/read/seek/close поверх
`File`). Никакого `convert_gif.py`, никакой пересборки — просто скинуть
`.gif` на карту.

## Подключение

MicroSD — та же отдельная SPI-шина (HSPI/SPI3), что в `firmware_sdcard`, не
общая с дисплеем (LovyanGFX держит SPI2 монопольно):

| Модуль | ESP32-S3 |
|---|---|
| GND | GND |
| VCC | **5V** (не 3V3!) |
| MISO | GPIO41 |
| MOSI | GPIO40 |
| SCK | GPIO39 |
| CS | GPIO38 |

**VCC — именно 5V.** На плате Catalex свой стабилизатор AMS1117-3.3 с
просадкой ~1В — при питании от 3.3V на выходе получается ~2.0-2.3V, SD-карте
не хватает для стабильной инициализации. Линии CS/SCK/MOSI/MISO сидят на
регулированных 3.3V через буфер на плате и безопасны для ESP32 в любом случае.

Опционально — сенсорная кнопка (TTP223, OUT→GPIO15): переключает экран
(GIF → сеть → календарь → GIF).

## Wi-Fi (для файлового менеджера)

Скопировать [src/secrets.h.example](src/secrets.h.example) в `src/secrets.h` и заполнить:

```cpp
#define WIFI_SSID_1 "приоритетная_сеть"
#define WIFI_PASS_1 "пароль"
#define WIFI_SSID_2 "резервная_сеть"     // например, точка доступа телефона
#define WIFI_PASS_2 "пароль"
```

(`secrets.h` в .gitignore). Без Wi-Fi проигрыватель тоже работает — сеть
только для файлменеджера и календаря (точное время по NTP), не обязательна
для показа гифок. Если ни одна сеть не доступна при старте, повтор попытки —
раз в 30 сек, без блокировки воспроизведения. Текущий SSID и IP смотрите на
экране "Network" (сенсорная кнопка).

По умолчанию часовой пояс UTC+5 (`TZ_OFFSET_SEC` в `main.cpp`) — поправьте
под свой.

## Подготовка карты

Положить `.gif` файлы в папку `/gif` на карте (или прямо в корень, если
`/gif` нет — резервный вариант). **Важно:** файлы должны быть уже готовыми
GIF (палитра ≤256 цветов, как из `tools/convert_gif.py` в `firmware_gif`, или
любой стандартный GIF — размер кадра желательно 240x240, иначе гифка просто
отцентрируется, но большие/тяжёлые файлы будут декодироваться медленнее).
`.h`-массивы тут не нужны — это было только под встраивание в flash.

## Управление

- **BTN1** короткое нажатие — следующий GIF
- **BTN1** долгое нажатие (>600мс) — предыдущий GIF
- **BTN2** — подсветка
- **Сенсорная кнопка (GPIO15)** — переключить экран: GIF → сеть (SSID/IP) →
  календарь (дата/время по NTP) → обратно на GIF. На экранах сети/календаря
  воспроизведение гифки приостанавливается

## Файловый менеджер по Wi-Fi

Если сеть подключена, на IP устройства (см. экран "Network" или Serial при
старте) поднимается простая HTTP-страница на порту 80:

- список `.gif` на карте со ссылками "скачать"/"удалить"
- форма загрузки нового `.gif` (кладётся в `/gif` на карте)

Без авторизации — только для домашней сети, не выставляйте в интернет.
Если удалить файл, который проигрывается прямо сейчас, плеер переключится
на первый в списке.

## Особенности

- Карта проверяется каждые 1.5 сек, пока не примонтирована — можно вставить
  на лету. Если карту вынуть во время работы — тоже определяется и вернёт
  к экрану "NO SD CARD"
- Список `.gif` собирается один раз при монтировании карты (папка `/gif`, а
  если её нет — корень), дальше идёт циклическое воспроизведение по кругу
- Отрисовка кадров — 1-в-1 логика из `firmware_gif` (прозрачные пиксели не
  трогают предыдущий кадр в спрайте), просто источник байт другой
- В `GIFReadFile()` сохранён "уродливый воркэраунд" (комментарий из
  официального примера `AnimatedGIF/examples/ESP32-LGFX-SDCard-GifPlayer`) —
  seek() у SD-библиотеки перестаёт нормально работать, если дочитать файл
  до самого последнего байта, поэтому читаем на 1 байт меньше
- Больше не упираемся в размер flash-партиции — гифки на SD-карте могут
  занимать гигабайты вместо ~8MB, и добавление нового файла не требует
  перепрошивки вообще
- Экран календаря — порт `wCalendar()` из `firmware_widgets`, но время берётся
  через NTP (`nowDT()`), а не из компилятора-как-часов — там сети не было
