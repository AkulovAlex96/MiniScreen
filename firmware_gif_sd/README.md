# MiniScreen — проигрывание GIF с MicroSD

То же самое, что `firmware_gif`, но без встраивания во flash: гифки читаются
файлами прямо с карты через колбэки `AnimatedGIF` (open/read/seek/close поверх
`File`). Никакого `convert_gif.py`, никакой пересборки — просто скинуть
`.gif` на карту.

## Подключение

MicroSD — та же отдельная SPI-шина (HSPI/SPI3), что в `firmware_sdcard`, не
общая с дисплеем (LovyanGFX держит SPI2 монопольно):

| Модуль | ESP32-S3 |
|---|---|
| GND | GND |
| VCC | 3V3 |
| MISO | GPIO46 |
| MOSI | GPIO40 |
| SCK | GPIO39 |
| CS | GPIO38 |

Опционально — сенсорная кнопка (TTP223, OUT→GPIO15): переключает на
следующий GIF.

## Подготовка карты

Положить `.gif` файлы в папку `/gif` на карте (или прямо в корень, если
`/gif` нет — резервный вариант). **Важно:** файлы должны быть уже готовыми
GIF (палитра ≤256 цветов, как из `tools/convert_gif.py` в `firmware_gif`, или
любой стандартный GIF — размер кадра желательно 240x240, иначе гифка просто
отцентрируется, но большие/тяжёлые файлы будут декодироваться медленнее).
`.h`-массивы тут не нужны — это было только под встраивание в flash.

## Управление

- **BTN1** — пауза/продолжить
- **BTN2** — подсветка
- **Сенсорная кнопка (GPIO15)** — следующий GIF

## Особенности

- Карта проверяется каждые 1.5 сек, пока не примонтирована — можно вставить
  на лету. Если карту вынуть во время работы — тоже определяется и вернёт
  к экрану "NO SD CARD"
- Список `.gif` собирается один раз при монтировании карты (папка `/gif`, а
  если её нет — корень), дальше идёт циклическое воспроизведение по кругу
- Отрисовка кадров — 1-в-1 логика из `firmware_gif` (прозрачные пиксели не
  трогают предыдущий кадр в спрайте), просто источник байт другой
- В `GIFReadFile()` сохранён "уродливый воркэраунд" (комментарий из
  официального примера `AnimatedGIF/examples/ESP32-LGFX-SDCard-GifPlayer`) —
  seek() у SD-библиотеки перестаёт нормально работать, если дочитать файл
  до самого последнего байта, поэтому читаем на 1 байт меньше
- Больше не упираемся в размер flash-партиции — гифки на SD-карте могут
  занимать гигабайты вместо ~8MB, и добавление нового файла не требует
  перепрошивки вообще
