# MiniScreen — инфо о MicroSD

Проверяет, вставлена ли MicroSD, и если да — показывает круговой прогресс-бар
занятого места, объём (использовано/всего, ГБ), количество файлов (рекурсивно
по всем папкам) и тип карты.

## Подключение

**CATALEX MicroSD Card Adapter** — своя, отдельная от дисплея SPI-шина
(HSPI/SPI3). Дисплей на LovyanGFX держит SPI2 монопольно, делить один
физический SPI-хост между LovyanGFX и библиотекой `SD` без низкоуровневой
возни с esp-idf нельзя — поэтому второй набор пинов, а не переиспользование
SCK/MOSI дисплея.

| Модуль | ESP32-S3 |
|---|---|
| GND | GND |
| VCC | **5V** (не 3V3!) |
| MISO | GPIO41 |
| MOSI | GPIO40 |
| SCK | GPIO39 |
| CS | GPIO38 |

**VCC — именно 5V, не 3.3V.** На плате Catalex стоит свой стабилизатор
AMS1117-3.3 — ему нужен запас по напряжению (dropout ~1В), при питании
ровно от 3.3V на выходе получается ~2.0-2.3V, чего SD-карте не хватает для
стабильной инициализации (карта отвечает на часть команд, но не проходит
ACMD41 до конца — путается с "картой не работает"). Буфер-микросхема на
плате сидит на регулированных 3.3V, так что линии CS/SCK/MOSI/MISO остаются
в 3.3V-домене и безопасны для ESP32 независимо от того, что подано на VCC.

Опционально — сенсорная кнопка (TTP223-типа, 3 пина VCC/GND/OUT):

| Модуль | ESP32-S3 |
|---|---|
| VCC | 3V3 |
| GND | GND |
| OUT | GPIO15 |

(GPIO15 на будущей плате зарезервирован под RST тач-панели CST816S — если
дойдёте до неё, освободите этот пин)

## Управление

- **BTN1** или **касание сенсорной кнопки** — пересканировать карту прямо сейчас
- **BTN2** — подсветка

## Особенности

- Карта проверяется каждые 1.5 сек, пока не примонтирована — можно вставить
  карту на лету без перезагрузки. Если карту вынуть во время работы, это
  тоже определяется (`SD.cardType() == CARD_NONE`) и экран уйдёт в "NO SD CARD"
- Объём/занято — из `SD.totalBytes()`/`usedBytes()` (FAT-таблица, без обхода
  дерева файлов), количество файлов — рекурсивным обходом (`SCAN_DEPTH_LIMIT`
  в `main.cpp`, по умолчанию 6 уровней вложенности)
- Отдельно считаются `.gif` — задел под проигрывание гифок с карты

## Что ещё можно посмотреть / сделать дальше

- **Список файлов** (уже собирается при сканировании, просто не выводится на
  экран — можно листать BTN'ами)
- **Размер сектора** (`SD.sectorSize()`, `SD.numSectors()`) — для диагностики
- **Проигрывание GIF прямо с карты** — да, вполне реально: библиотека
  `AnimatedGIF` (уже используется в `firmware_gif`) поддерживает открытие по
  имени файла через колбэки `open/read/seek/close` поверх `File` — есть готовый
  референс в её примерах: `AnimatedGIF/examples/ESP32-LGFX-SDCard-GifPlayer`.
  Плюс перед конвертером: гифки на SD не занимают flash и не требуют
  перепрошивки/пересборки при добавлении новых — то, во что как раз упёрся
  `firmware_gif` (партиция на 8MB уже наполовину занята тремя гифками).
  Формат хранения — обычный `.gif` файл как есть, `.h`-массивы тут не нужны
  (это было решение только для встраивания в flash).
