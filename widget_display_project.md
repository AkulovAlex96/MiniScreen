# Проект: Настольный виджет-экран на ESP32-S3

Небольшой настольный девайс с дисплеем и настраиваемыми виджетами: часы, погода, акции, опрос сервера и т.д. Своя печатная плата, аккумуляторное питание, потенциально мелкая серия под продажу.

## Концепция

- **Экран:** ~35 мм (1.69" IPS 240×280 или круглый 1.28" 240×240)
- **Управление:** ёмкостный тач (жесты в LVGL, свайпы между виджетами)
- **Питание:** Li-ion/LiPo + USB-C для зарядки и прошивки
- **Фичи:** RGB-подсветка по периметру, автояркость по фоторезистору, Wi-Fi для данных
- **UX:** виджеты добавляются/настраиваются через встроенный UI (LVGL)

---

## Микроконтроллер

**ESP32-S3-WROOM-1-N16R8**

- Двухъядерный Xtensa 240 МГц
- 512 КБ SRAM встроенной
- **16 МБ Flash + 8 МБ PSRAM** (буфер экрана уходит в PSRAM, память с запасом)
- Нативный USB (D+/D− прямо на GPIO19/20, без CH340/CP2102)
- Wi-Fi + BLE
- Готовый модуль с антенной, сертифицирован, есть в базовой библиотеке JLCPCB
- Цена ~350–500₽

**Занятые GPIO (не трогать):** 26–37 заняты под flash/PSRAM на N16R8 (octal PSRAM)

### Обвязка ESP32-S3-WROOM-1

**Питание модуля:**
- 3V3 (пин 2) → C 22мкФ **0805** на GND + C 100нФ **0603** на GND (ставить максимально близко к модулю)
- GND (пины 1, 40, EP) → GND

**Кнопка BOOT (GPIO0):**
- GPIO0 (пин 27) → тактовая кнопка → GND
- GPIO0 → R 10 кОм **0603** → 3V3 (pull-up, чтобы не висел в воздухе при нормальной работе)

**Кнопка RESET (EN):**
- EN (пин 3) → тактовая кнопка → GND
- EN → R 10 кОм **0603** → 3V3 (модуль уже имеет внутренний pull-up, внешний необязателен, но рекомендуется)
- EN → C 100нФ **0603** → GND (подавляет дребезг, обеспечивает надёжный ресет при старте)

**ESD-защита USB (USBLC6-2SC6):**
- I/O1 → USB_DP, I/O2 → USB_DN
- VBUS (пин 4) → VBUS_5V (через R 0 Ом или напрямую)
- GND (пин 2) → GND
- Ставить максимально близко к USB-C разъёму, до любых других цепей на D+/D−

**Автозагрузка (autoreset через USB DTR/RTS):**
- Нативный USB CDC ESP32-S3 сам обрабатывает DTR/RTS → RESET/BOOT без внешней схемы Q1/Q2 (как у CH340). Внешние схемы autoreset не нужны.

## Дисплей

**Основной выбор:** голая панель **1.69" ST7789 240×280 IPS** с ёмкостным тачем **CST816S** (общий или отдельный FPC-шлейф).

**Альтернатива:** круглая **GC9A01 1.28" 240×240** — можно сделать универсальную плату с двумя FPC-разъёмами (голые панели требуют разной распиновки, единый разъём не выйдет).

### ST7789 — подключение по SPI

| Сигнал | GPIO | Примечание |
|--------|------|------------|
| CS | 10 | Chip Select |
| SCK | 11 | SPI Clock |
| MOSI | 12 | Data (SPI) |
| DC | 13 | Data/Command |
| RST | 14 | Reset (активный низкий) |
| BL | 21 | Подсветка (PWM) |
| VCC | 3V3 | C 100нФ **0603** + C 1мкФ **0603** на GND рядом с FPC |
| GND | GND | |

**FPC ZIF-разъём:** партнамбер выбирать после покупки конкретной панели — количество и шаг контактов зависят от шлейфа.

### Драйвер подсветки

**Вариант A — параллельные LED на 3.3V (проще):**
```
GPIO21 (PWM) → Gate MOSFET AO3400
3V3 → Drain AO3400 → R токозадающий → LED+ панели → LED− → GND
```
Токозадающий резистор: Vf у белых/синих LED ~3.0–3.2V. При 3.3V и 20 мА: R = (3.3 − 3.1) / 0.020 ≈ 10 Ом. Подобрать по конкретной панели.

**Вариант B — последовательные LED (boost, точнее):**
AW9364 или SGM3732 — I2C-управляемый boost-драйвер, контролирует ток напрямую. Нужен, если у панели 2–4 последовательных BL LED (Vf суммарно > 3.3V). Подключение: VIN → SYS, VOUT → LED+, EN/PWM → GPIO21, SCL/SDA → I2C.

### CST816S — ёмкостный тач (I2C)

| Сигнал | GPIO | Примечание |
|--------|------|------------|
| SDA | 8 | I2C (с pull-up 4.7к к 3V3 — общий для всей шины) |
| SCL | 9 | I2C (аналогично) |
| INT | 47 | Прерывание по касанию (активный низкий, pull-up 10к **0603** к 3V3) |
| RST | 15 | Сброс чипа (активный низкий, тянуть к 3V3 через R 10к **0603**, ESP32 держит HIGH) |
| VDD | 3V3 | C 100нФ **0603** на GND рядом |
| GND | GND | |

I2C адрес CST816S: `0x15`.
Второй FPC ZIF-разъём — если тач на отдельном шлейфе от дисплея.

**Что появляется на плате из-за голой панели:**
- FPC ZIF-разъём под шлейф дисплея
- Второй FPC под тач (если шлейф отдельный)
- Драйвер подсветки (MOSFET + R или AW9364)
- Обвязочные конденсаторы по питанию дисплея (100нФ + 1мкФ)

**LVGL** для UI, **SquareLine Studio** как визуальный редактор.

## Питание и заряд

Общая топология питания:

```
VBUS_5V ──► BQ24074 ──► SYS ──► MT2492 ──► 3V3
               │
               └──► VBAT (аккум JST-PH 2.0)
                        │
                        └──► MAX17048 (fuel gauge, I2C)
```

Power path BQ24074: когда USB подключён — SYS питается от VBUS и одновременно заряжает аккум. Без USB — SYS питается от аккума. Переключение автоматическое, без прерывания питания.

---

### USB-C разъём

**TYPE-C-31-M-12** (16-pin, HRO/Korean Hroparts) — есть в basic-библиотеке JLCPCB.

| Пин разъёма | Сигнал | Куда |
|-------------|--------|------|
| A4, B4 | VBUS | VBUS_5V (объединить) |
| A5, B5 | GND | GND (объединить) |
| A6, B6 | D+ | USB_DP (объединить) |
| A7, B7 | D− | USB_DN (объединить) |
| A8 | CC1 | R 5.1 кОм **0603** → GND |
| B8 | CC2 | R 5.1 кОм **0603** → GND (отдельный резистор!) |
| A9, B9 | SBU1, SBU2 | NC |
| EH1–EH4 | Shield | GND |

**CC-резисторы 5.1 кОм** — сигнализируют хосту что это UFP (device), запрашивают 5V/500mA–3A в зависимости от кабеля. Без них зарядка может не запустится.

**USBLC6-2SC6** — ставить сразу после разъёма, до всего остального:
```
USB_DP ──► I/O1 ┐
USB_DN ──► I/O2 ┤ USBLC6-2SC6
VBUS   ──► VBUS ┤
GND    ──► GND  ┘
```

---

### BQ24074 (заряд + power path)

| Пин | Куда | Номинал |
|-----|------|---------|
| IN (13) | VBUS_5V | + C 1мкФ **0603** на GND |
| OUT (10, 11) | SYS | объединить, + C 4.7мкФ **0805** на GND |
| BAT (2, 3) | VBAT | объединить, + C 10мкФ **0805** на GND |
| VSS (8), EP (17) | GND | |
| ILIM (12) | R → GND | 1.6 кОм **0603** → I_lim ≈ 1.5 А |
| ISET (16) | R → GND | 3 кОм **0603** → I_chg ≈ 300 мА |
| ITERM (15) | R → GND | 10 кОм **0603** → I_term ≈ 30 мА |
| TMR (14) | R → GND | 1.6 кОм **0603** → таймер ≈ 5 ч |
| TS (1) | делитель VBAT/GND | 10к/10к **0603** (без термистора) |
| CE# (4) | GND | заряд разрешён |
| EN1 (6), EN2 (5) | GND | режим по умолчанию |
| CHG# (9) | R 1к **0603** → LED синий → SYS | горит при заряде (активный низкий) |
| PGOOD# (7) | R 10к **0603** pull-up к **3V3** → GPIO | LOW = USB подключён |

**Формулы (из даташита):**
- `I_CHG = 890 / R_ISET(кОм)` → 890 / 3 = 297 мА ≈ 300 мА
- `I_TERM = I_CHG / 10` (при R_ITERM = R_ISET × 10) → 30 мА
- `I_ILIM = 2500 / R_ILIM(кОм)` → 2500 / 1.6 = 1562 мА ≈ 1.5 А

**PGOOD# подтягивать строго к 3V3, не к SYS** — SYS при зарядке может прыгать до 4.2V, GPIO ESP32 не 5V-tolerant.

### MT2492 (DC-DC buck SYS → 3.3V)

```
         L1 2.2мкГн 0805
SYS ──┬──[ SW(6) ]────────────┬── 3V3
      │                       │
  C 10мкФ 0805           C 22мкФ 0805
      │                       │
     GND              R12 180к 0603┤
                                   ├── FB(3)
                        R13 40.2к 0603
                                   │
                                  GND
```

| Пин | Куда | Примечание |
|-----|------|------------|
| IN (5) | SYS | + C 10мкФ **0805** на GND |
| EN (4) | IN через R 100к **0603** | + C 10нФ **0603** на GND (soft-start) |
| BS (1) | C 10нФ **0603** → SW(6) | bootstrap |
| SW (6) | L 2.2мкГн **0805** → 3V3 | |
| FB (3) | делитель R12/R13 | R12=180к **0603**, R13=40.2к **0603** |
| GND (2) | GND | |

**Проверка:** Vout = 0.6 × (1 + 180/40.2) = 0.6 × 5.478 = **3.287V ≈ 3.3V** ✓

**ВАЖНО:** L1 в EasyEDA — **2.2uH** (микрогенри), не 2.2mH!

| Пин | Куда | Примечание |
|-----|------|------------|
| VDD (3) | 3V3 | + C 1мкФ **0603** на GND рядом |
| CELL (2) | VBAT | измерение напряжения аккума |
| CTG (1) | GND | |

### Разъём аккума

**JST-PH 2.0 мм**, 2 пина:
- Пин 1 (красный) → VBAT
- Пин 2 (чёрный) → GND

Подходит LiPo/Li-ion 500–2000 мАч с одной ячейкой (1S, 3.7V номинал, 4.2V max).
При токе заряда 300 мА — минимально рекомендуемая ёмкость аккума 500 мАч (C/1.7).
Если ставить 18650 в холдер — холдер с тем же JST-PH выводом.

## RGB-подсветка

**SK6812MINI-E** (3.5×3.5 мм) или **SK6812 Side-View 4020** для edge-подсветки через корпус, 6–12 штук по периметру.

**Почему SK6812:** работает от **3.3V** (в отличие от WS2812, которым нужны 5V), питается прямо от 3V3-шины, уровень данных 3.3V от ESP32 подходит без level shifter.

### Схема цепочки

```
GPIO45 → Gate AO3400
3V3 → Drain AO3400 → VCC цепочки
Source AO3400 → GND

GPIO48 → R 330 Ом **0603** → DIN (LED1) → DOUT (LED1) → DIN (LED2) → ... → DOUT (LEDn)

Каждый LED: C 100нФ **0603** между VDD и LED_GND (как можно ближе к корпусу)
Начало цепочки: C 100мкФ **D4.0×H5.5 SMD электролит** между VCC и LED_GND (bulk, сглаживает броски тока)
```

**Потребление:** SK6812MINI-E при полном белом ~12 мА на LED, 12 штук = 144 мА. В реальном использовании значительно меньше — цвет, яркость. 3V3-шина держит до ~600–800 мА (зависит от MT2492 и дорожек), запас есть.

**MOSFET AO3400** (N-channel, SOT-23):
- Gate → GPIO45 (HIGH = LED включены)
- Drain → VCC цепочки SK6812
- Source → GND
- R 10 кОм **0603** Gate–Source (удерживает выключенным при неинициализированном GPIO)

**LED_OUT разъём (HDR-M-2.54_1x3)** — для расширения внешней цепочки:
| Пин | Сигнал |
|-----|--------|
| 1 | 3V3 |
| 2 | DOUT от последнего LED (U15) |
| 3 | LED_GND |

Внешние LED подключаются через MOSFET (тот же LED_GND) — включаются и выключаются вместе с основной цепочкой.

Софт: FastLED или встроенный RMT-драйвер ESP32 (предпочтительнее — не тянет стороннюю либу).

## Датчики (опционально)

### Фоторезистор GL5528 (автояркость)

Делитель напряжения между 3V3 и GND, средняя точка → ADC:

```
3V3 ──── R_pull 10 кОм 0603 ──┬── GPIO4 (ADC1_CH3)
                               │
                             GL5528
                               │
                              GND
```

- **GL5528:** сопротивление 10–20 кОм при комнатном освещении, 1 МОм в темноте
- **R_pull 10 кОм 0603:** при ярком свете ADC близко к 0V (фоторезистор ~1кОм), в темноте близко к 3.3V. Инвертированная зависимость — учесть в коде
- **C 100нФ 0603** на GPIO4 → GND: подавляет наводки на длинной дорожке к ADC
- Физически ставить у края платы или под вырезом корпуса — нужен доступ к свету

**В коде:** `analogRead(4)` → нормировать в 0–100%, применить к PWM подсветки через кривую (не линейно, а логарифмически — глаз нелинейный).

---

### Qwiic / JST-SH 4-pin (внешние I2C датчики)

Стандарт SparkFun Qwiic — JST-SH 1.0 мм, 4 пина:

| Пин | Сигнал | Цвет провода |
|-----|--------|--------------|
| 1 | GND | чёрный |
| 2 | 3V3 | красный |
| 3 | SDA | синий |
| 4 | SCL | жёлтый |

- SDA, SCL — на общую I2C шину (GPIO8/9), pull-up уже есть на плате
- 3V3 → с 3V3-шины, ток на внешний датчик обычно < 20 мА — нормально
- Можно поставить прямо на плату или вывести на боковой торец для подключения внешнего модуля

Совместимые датчики: SCD40 (CO₂), SHT31 (температура/влажность), BMP280 (давление), VEML7700 (люксметр) и тысячи других на I2C.

---

### Пищалка (опционально)

Пассивный зуммер (passive buzzer) 5×5 мм SMD:
- «+» → GPIO через R 100 Ом **0603** → зуммер → GND
- Управление PWM: частота определяет тон, скважность — громкость
- Или транзистор NPN BC817 если зуммер требует > 10 мА

Вибромотор (опционально): ERM coin-motor, N-MOSFET AO3400 + диод защитный (fly-back) + R 10 Ом **0603** в gate.

## I2C шина (общая)

Все I2C устройства на одной шине GPIO8 (SDA) / GPIO9 (SCL):

| Устройство | Адрес | INT/RST пин |
|------------|-------|-------------|
| MAX17048 (fuel gauge) | `0x36` | ALRT# → GPIO (TBD) |
| CST816S (тач) | `0x15` | INT → GPIO47, RST → GPIO15 |
| AW9364 (BL driver, если используется) | `0x36` / `0x37` | — |
| Qwiic разъём | зависит от модуля | — |

**Важно:** MAX17048 и AW9364 могут конфликтовать по адресу `0x36` — уточнить по даташиту выбранного BL-драйвера. Если конфликт — использовать MOSFET-вариант подсветки без I2C.

**Pull-up резисторы:**
- 1 комплект **4.7 кОм 0603** на SDA и SCL к 3V3
- Ставить ближе к ESP32, не у каждого устройства отдельно — несколько параллельных pull-up занижают суммарное сопротивление и ускоряют фронты (может мешать на 400 кГц)
- Если Qwiic-модуль принесёт свои pull-up — суммарное будет ~2.4 кОм, допустимо

**Скорость:** стандарт 100 кГц (Standard Mode) достаточен. При трассировке дорожки I2C держать короткими (< 10 см), без переходных отверстий посередине если возможно.

## Test Points (для отладки)

**Формат:** круглые SMD-pad **1.5 мм** (щуп мультиметра), **2 мм** для частых точек (VBAT, GND). Подписи на silkscreen. Группировать по блокам рядом с соответствующей микросхемой.

### Питание

| TP | Сигнал | Что мерить | Норма |
|----|--------|------------|-------|
| TP1 | VBUS | напряжение USB | 4.5–5.5V при подключении |
| TP2 | SYS | выход BQ24074 | 3.5–4.2V (от аккума) или 4.4V (от USB) |
| TP3 | VBAT | напряжение аккума | 3.0–4.2V |
| TP4 | 3V3 | выход MT2492 | 3.25–3.35V |
| TP5–7 | GND | земля | 0V — по краям платы ×3 |

### Buck MT2492 (диагностика)

| TP | Сигнал | Что смотреть |
|----|--------|--------------|
| TP8 | SW | прямоугольник ~1.2 МГц на осциллографе (если нет — чип не работает) |
| TP9 | FB | строго **0.60V** постоянка (если нет — неправильный делитель R12/R13) |
| TP10 | EN | = IN (SYS) при нормальной работе |

### Зарядник BQ24074

| TP | Сигнал | Что смотреть |
|----|--------|--------------|
| TP11 | CHG# | LOW при заряде, HIGH когда заряжен или нет USB |

| TP | Сигнал | Что смотреть |
|----|--------|--------------|
| TP14 | SDA | при активном обмене — пачки импульсов. В покое = HIGH (3.3V) |
| TP15 | SCL | при активном обмене — меандр 100 кГц. В покое = HIGH (3.3V) |

Если SDA/SCL в покое = LOW — устройство на шине держит линию (I2C lockup), нужен RESET.

### USB

| TP | Сигнал | Зачем |
|----|--------|-------|
| TP16 | USB_DP | если прошивка не заливается — смотреть перекрёстно с USB_DN |
| TP17 | USB_DN | дифпара, должны быть симметричны |

### Отладочная «шапка» (опционально)

JST-SH 6-pin или 2.54 мм гребёнка на краю платы:

| Пин | Сигнал |
|-----|--------|
| 1 | GND |
| 2 | 3V3 |
| 3 | UART TX (GPIO43) |
| 4 | UART RX (GPIO44) |
| 5 | BOOT (GPIO0) |
| 6 | RESET (EN) |

Позволяет прошивать и смотреть логи через USB-UART адаптер как резервный канал, если нативный USB не работает.

## KiCad Python API (заметка на потом)

Два API:
- **pcbnew** (SWIG, старый) — работает только с платой (.kicad_pcb), не со схемой. Так сделаны плагины KiKit, InteractiveHtmlBom
- **kipy / IPC API** (KiCad 9+) — новый, стабильный, общается с живым KiCad через сокет. Через него можно было бы автоматизировать разводку из Claude Code
- **kicad-cli** — консольные команды: экспорт PDF/герберов, ERC, DRC. Полезно для CI и скриптов

Применение: расстановка RGB-LED по периметру платы скриптом, автоматизация проверок, генерация артефактов для JLCPCB.

## Полезные ссылки и заметки

- **JLCPCB PCBA** — базовая библиотека (basic parts) дешевле в сборке. Все ключевые компоненты BOM подобраны с прицелом на неё
- **LCSC** — покупка чипов, есть даташиты по партнамберам
- **SquareLine Studio** — редактор UI для LVGL
- **FastLED** или ESP32 RMT-driver — для SK6812
- **EasyEDA (Standard)** — проще для JLCPCB, компоненты со склада сразу с футпринтами и 3D-моделями
- **KiCad** — если нужен полный контроль и Python-автоматизация
